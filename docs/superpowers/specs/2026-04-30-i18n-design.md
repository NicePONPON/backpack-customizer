# Bilingual Site (English / Traditional Chinese) — Design

**Date:** 2026-04-30
**Status:** Approved (pending spec review)

## Goal

Allow visitors to switch the marketing site and customize tool between English and Traditional Chinese (`zh-TW`). The toggle is manual; English is the default. The `/invoice` route stays English-only as a deliberate, time-bounded exclusion.

## Non-goals

- IP-based geolocation or auto-redirect.
- Simplified Chinese (`zh-CN`).
- Translating `/invoice`, `/api/*`, `pricing.ts` currency codes, or the dev-only `?calibrate=*` panels.
- SEO-friendly per-locale URLs (no `/zh-TW/...` route prefix).

## Decisions Made

| # | Question | Choice |
|---|---|---|
| 1 | Scope of first delivery | Marketing pages + customize tool. Skip `/invoice`. |
| 2 | URL strategy | Cookie only. Same URLs serve both languages. |
| 3 | Translation authoring | Claude translates all strings; user edits anything that reads off after build. |
| 4 | Detection | None. English default. Manual toggle only. |
| 5 | Color names | Translated to Chinese marketing names (Claude drafts, user can refine). |
| 6 | Lookbook ad images | Auto-discovered from `public/advertisement/` by `EN-` / `TW-` filename prefix. |

## Architecture

### Library

`next-intl` configured as a provider only — no routing integration. We get its typed `useTranslations()` hook, dev warnings for missing keys, and clean server / client component support without paying for the full middleware + `[locale]` segment restructure.

### Locale state

- Stored in a `NEXT_LOCALE` cookie (`max-age=31536000`, `path=/`, no domain restriction).
- Values: `en` or `zh-TW`. Anything else falls back to `en`.
- Read once per request by `src/app/layout.tsx` (a server component) via `cookies()` from `next/headers`. Passed to `NextIntlClientProvider` along with the matching messages bundle.

No middleware. No filesystem restructure. No `[locale]` route segment.

### Toggle UI

`src/components/LanguageToggle.tsx`, client component:

- Two-segment pill: `EN` / `中`. Active segment is filled white (`rgba(255,255,255,0.95)` background, `#111` text); inactive is transparent with a subtle border.
- Total dimensions ~56×28px. Glass-card border consistent with the rest of the chrome.
- On click: writes `NEXT_LOCALE` cookie via `document.cookie`, then `window.location.reload()`. The full reload is intentional — locale is read server-side at render, and a SPA-style swap would need a Context refresh dance not worth the complexity for chrome-level UI.

Mounted by `SiteHeader` with absolute positioning:

```css
position: absolute;
top: 50%;
right: max(20px, env(safe-area-inset-right));
transform: translateY(-50%);
```

The logo stays centered (`justifyContent: center` on the header flex container is unchanged).

### Translation files

Two JSON bundles in `src/i18n/messages/`:

```
src/i18n/messages/en.json
src/i18n/messages/zh-TW.json
```

Namespaced by surface so editing maps cleanly to the file each string lives in:

| Namespace | Owns |
|---|---|
| `nav` | SubNav links (Home, Lookbook, Customize, Shop) |
| `header` | Toggle pill labels, aria-labels |
| `footer` | Copyright line, contact text |
| `intro` | "Double-tap to skip" hint |
| `share` | "Link copied" toast, share-dock aria-labels |
| `home.hero` | Hero tagline + supporting copy |
| `home.whyThisBag` | Feature card titles + body |
| `gallery` | Section header, "Coming Soon", notify-me labels, error text |
| `shop` | Shop placeholder copy |
| `customize.sections` | "Color", "Embroidery", "Zipper", "Size" group headings |
| `embroidery` | Control labels, color picker, line count, position labels |
| `size` | "14 inch" / "16 inch" / size visualizer comparison label |
| `colors` | 5 brand color names (Pine Smoke, Tideglass Blue, Olive Cream, Butter Glow, Ivory Dune) |
| `parts` | ~15 bag-region labels (Front Top, Back Main, Side, etc.) |
| `summary` | Summary table headers, total labels |

**Authoring:** Claude writes both `en.json` and `zh-TW.json` in one pass. The user reviews the rendered site and edits any string in `zh-TW.json` that reads off-tone for the Taiwan audience.

**Missing-key fallback:** the `NextIntlClientProvider` is configured with both `messages` (the active locale) and a `getMessageFallback` callback that looks the missing key up in the English bundle and returns it. So a half-finished `zh-TW.json` never renders blank — any missing key resolves to its English string until the gap is filled. The English bundle is bundled into the client payload for both locales (small cost; ~4-8 KB minified+gzipped) so the fallback works without an extra round trip.

### Lookbook advertisement auto-discovery

The `/gallery` ad rail currently hardcodes a list of three filenames. Under this design, the rail enumerates `public/advertisement/` at render time and shows whichever PNGs match the active locale's prefix.

**File convention:**
- `EN-*.png` — shown when locale is `en`
- `TW-*.png` — shown when locale is `zh-TW`
- Sort is alphabetical, so `TW-01-Hero.png` orders before `TW-02-Comparison.png`. Add a numeric prefix to control display order.

**Server utility** at `src/lib/loadAdvertisements.ts`:

```ts
import fs from "node:fs";
import path from "node:path";

export function loadAdvertisements(locale: "en" | "zh-TW"): string[] {
  const prefix = locale === "zh-TW" ? "TW-" : "EN-";
  const dir = path.join(process.cwd(), "public/advertisement");
  return fs.readdirSync(dir)
    .filter(f => f.startsWith(prefix) && f.toLowerCase().endsWith(".png"))
    .sort()
    .map(f => `/advertisement/${f}`);
}
```

**Component split:** today's `AdvertisementPanel` (client, in `src/app/gallery/page.tsx`) is split into:
- `AdvertisementPanel` (server component) — calls `loadAdvertisements(locale)`, passes the array down.
- `AdRail` (client component) — receives the URL list and runs the existing scroll-snap / scale / progress-bar interactivity.

This keeps the gallery `page.tsx` light and lets the file enumeration happen on the server side without a "use client" boundary getting in the way of `fs.readdirSync`.

**Adding new ads:** drop `EN-NewSlide.png` + `TW-NewSlide.png` into `public/advertisement/`, commit the assets, and they appear next render. No code edit needed.

**Asymmetry handling:** if only the EN counterpart exists for a slot (e.g., the user paints `EN-Promo.png` but hasn't done `TW-Promo.png` yet), the zh-TW rail simply has one fewer card. There is no fallback that would mix EN graphics into a TW rail — the prefix is the source of truth.

### Routes

| Route | Locale-aware? | Notes |
|---|---|---|
| `/` (home) | Yes | Wraps `HomePage` in provider via root layout. |
| `/gallery` | Yes | Server-renders the ad list; client `Gallery` reads its labels via `useTranslations()`. |
| `/customize` | Yes | All client-component strings swap. Color names come from `colors` namespace. |
| `/shop` | Yes | Whatever placeholder ships uses translated copy. |
| `/invoice` | **No** | Hardcoded English. Cookie is set but unused on this route. The customize → invoice link drops locale (`/invoice?d=...`). |
| `/api/*` | N/A | API responses stay English. |
| `/opengraph-image` | N/A | Static painted-bag PNG, language-agnostic. |

## Data Flow

1. Browser request hits Next.js.
2. Root layout (`src/app/layout.tsx`) reads `NEXT_LOCALE` cookie. Resolves to `"en"` or `"zh-TW"` (defaulting to `en`).
3. Loads matching message bundle from `src/i18n/messages/`.
4. Wraps page tree in `<NextIntlClientProvider locale={locale} messages={messages}>`.
5. Server components call `getTranslations()`. Client components call `useTranslations()`. Both resolve against the same bundle.
6. The `/gallery` server page passes `locale` to `loadAdvertisements()`, which reads `public/advertisement/` and returns the matching URL list.
7. `LanguageToggle` (client) writes the cookie + reloads. The next request goes through steps 1-6 with the new locale.

## Error Handling

- **Missing zh-TW key:** falls back to English string via `getMessageFallback`. Logged to console in dev.
- **Malformed cookie value:** treated as `en`.
- **No `EN-*.png` files in `public/advertisement/`:** ad rail renders nothing. Gallery section header still appears. Acceptable — this is dev-only state.
- **`fs.readdirSync` failure** (path doesn't exist on the server): catch, log, return `[]`. Gallery degrades to "no ads" rather than crashing the page.

## Testing

No automated test suite exists (per CLAUDE.md). Manual verification on Vercel preview:

- **Toggle works on every page:** `/`, `/gallery`, `/customize`, `/shop` — click `中` → page reloads in Chinese, click `EN` → reloads in English. Cookie persists across navigation.
- **Cookie survives across browser sessions:** close + reopen browser, language preference sticks.
- **Gallery ads switch:** in EN mode, `EN-*.png` files render; in zh-TW mode, `TW-*.png` files render. Adding a new pair without code change updates both rails.
- **`/invoice` stays English:** even with `NEXT_LOCALE=zh-TW`, `/invoice?d=...` renders English.
- **Customize tool:** color names, part labels, embroidery controls, summary table all swap. Generated invoice link still works (English).
- **Missing-key fallback:** delete a key from `zh-TW.json`, verify the corresponding English string renders without a console error fatal.
- **Mobile chrome:** toggle is reachable and tappable on iPhone (notch / safe-area), doesn't overlap the centered logo on narrow viewports (375px and up).

## Out-of-Scope Follow-Ups

These are deliberately deferred:

1. **`/invoice` translation** — separate sub-project. Includes the bag preview labels, line item descriptions, currency formatting, and the BillTo form. Not trivial because the invoice is captured into a PDF via `html2canvas`.
2. **Simplified Chinese (`zh-CN`)** — would add a third translation bundle and a third toggle option. Only relevant if the China market becomes a target.
3. **URL-based locale (`/zh-TW/...` prefix)** — adds shareability of language-locked links. Requires a `[locale]` route segment restructure.
4. **IP-based default detection** — if data shows TW visitors aren't finding the toggle, revisit auto-detection with a Vercel header check at the layout level (not middleware).
